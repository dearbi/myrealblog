计科中外学习指南

中外第三学期课程指南（大二上）

  1. 操作系统 ：为考研 408 课程，请自己考虑是不是要好好学一下。
  2. 离散数学 ：为计算机科学基础，难度较大，建议上课好好听，做好笔记、好好复习。
  3. 面向对象开发 ：授课语言为 Java，难度较大，之前高级程序设计没学明白的建议提前花时间复习一下；核心是面向对象的思想。

参考资料

  1. 操作系统 ：Operating Systems: Three Easy Pieces ，对应中文版见群文件
  2. 计算机系统 ：《深入理解计算机系统 - CSAPP》，建议像在计算机领域发展的同学阅读一下，见群文件
  3. 离散数学 ：《离散数学》左孝凌著，见群文件，同时该书为下学期教材
  4. 离散数学浓缩版 ：我的博客 ，内容不全，可能有错误，但是浓缩，欢迎勘误
  5. 面向对象开发 ：核心思想可以阅读《大话设计模式》自行体会（该书示例代码不是 Java），见群文件
  6. 面向对象开发 ：《Java 编程思想》，见群文件

中外第四学期课程指南（大二下）

  1. 数据结构 ：为考研 408 课程，请自己考虑是不是要好好学一下。
  2. 计算机网络 ：为考研 408 课程，请自己考虑是不是要好好学一下。
  3. 智能系统 ：李老师授课，主题为人工智能，22 级无参考意义，他们学的跟你们不一样。
  4. 事件驱动程序设计 ：李老师授课，主题为 C++ QT，主要进行 C++ 面向对象的学习。
  5. 本学期外文专业课：数据结构、智能系统、事件驱动程序设计，事件驱动不确定是否有外教授课，前两门有外教授课。

学习指南

  1. 计算机网络 ：郑强老师授课，日常有实验 + 报告，个人认为讲的比较清晰，但是计算机网络整体上体量大、内容杂、知识点多，还是要花些时间学习的。
  2. 数据结构 ：肖爱梅老师 + 外教授课，课程本身难度大，课时紧张，如果平常不做练习，很难不挂科，学习的秘籍就是多敲代码、多敲代码、多敲代码！
  3. 智能系统 ：李静老师 + 外教授课，课程本身难度尚可，有实验，李老师授课很清晰有条理，但是需要时间适应授课风格。
  4. 事件驱动程序设计 ：李静老师，c++ 面向对象和 Java 有所区别，且很少涉及 QT 的使用，但是实验 + 大作业必须要使用 QT 完成，需要大家自学如何使用。
  5. 关于自学 ：我想大家经过了一年多的大学生活也有所体会了，有些同学对课内的教学 “嗤之以鼻”，但还有部分同学认为课内教授的内容和考核相比仍有差距。在培养方案中本身就具有对 “自主学习的能力” 的要求，因此希望大家能理解自学的意义，并且能够找到正确的方法来借此不断提升自己的能力。

关于 AI

应该有相当的同学在学习中使用 AI 了吧，但是也希望大家能够正视其作用和摆正其定位，将其用于生产生活自然是生产力的提高，但如果期望其完全替代自己的学习任务那就偏离了正确的用途甚至可能涉及了学术不端。在日常学习中还是非常建议大家用传统方式（书籍、视频、文档、博客）与 AI 结合的方式进行学习，个人体会这样做的学习效率是很高的，我们使用 AI 辅助的目的是为了掌握相关的技术。坦白说现在的 AI 在完成简单任务上表现很不错，但在完成复杂任务上仍处在辅助的位置。如果我们自身没有掌握前置的 “简单技术”，非常悲观的是大家可能连使用 AI 执行复杂任务都做不到。
现在可能有同学因为使用了 AI 技术而受益，也在某一方面短暂的取得了优势，那不妨思考一下 AI 在自己的学习生活中到底扮演了什么角色，作为个人核心的学习能力、编码技术、体系思维是进步了还是退化了？“宁拙毋巧，功不唐捐”，这是 SDUTACM 的口号，和大家共勉。
关于 AI 的伦理问题也是大家热议的问题，以上也仅是我的一些看法，欢迎大家多多思考交流！

中外第五学期课程指南（大三上）

  1. 编译原理 ：无纸质考试，平时上机测试计入总成绩。
  2. 数据库系统 ：双语课，换老师了，不清楚有无线下外教。
  3. 软件测试与检验 ：双语课，不清楚有无外教。
  4. 实训 ：共 3 个，持续 7 周，数据库有企业授课，数据结构未知，编译原理实训无授课。

学习指南

  1. 编译原理 ：内容较多，但是阶段测试，上课最好听一下，按时完成平时练习 + 考前突击挂科很难。
  2. 数据库 ：主要学习 SQL，难度不大，可以提前熟悉。
  3. 软件测试与检验 ：请按照老师实际要求学习。
  4. 实训忠告 1 ：数据库实训为 Web 项目，小组完成。建议提前学习 HTML+CSS+JS 的前端基础，可以扩展学习 Vue.js（建议 Vue3 组合式），后端可以使用 SpringBoot（Java）、Laravel（PHP）、Django（Python）等框架，企业培训内容为 SpringBoot。
  5. 实训忠告 2 ：对于编译原理实训，个人完成，不组队。从最终作品完成度来评价，完成度越高对于代码水平以及编译原理基础要求越高，但是达到及格线对于个人能力要求并不是很高。该实训根据个人最终目标，高完成度一定会用到汇编。
  6. 关于汇编 ：第六学期微机原理会教汇编，在编译原理中没有要求，属于自己拓展的范畴，不超纲。推荐下方的参考资料，可以挑着看，配合代码一起完成。
  7. 其他 ：数据结构要求未知，以老师要求为准。

参考资料

  1. 前端学习资料 ：尚硅谷前端入门 html+css 零基础教程 、尚硅谷 JavaScript 基础 & 实战
  2. 编译原理 ：（建议优先听讲，王海鹏老师讲的很清晰）教材（紫书）以及 “龙书”（黑皮书），见群文件
  3. 汇编语言 ：《汇编语言》王爽著，见群文件
  4. 汇编编程环境 1 ：汇编语言学习 DOSBox+MASM 安装及使用教程
  5. 汇编编程环境 2 ：使用 VScode 进行 8086 汇编学习

期末数据库实训 Tips

考察前后端开发，后续的软件实训，项目实训均为此形式。
前端学习建议 ：

  1. 先学 html css js 的前端基础
  2. 再学 vue.js 框架开发

后端学习建议 ：

  1. 如果想要在实训期间跟着老师学习，建议复习 Java 以及学习 spring boot
  2. 如果想要快速上手，推荐基于 Python 的 flask 或 django，flask 上手难度显著低于 django
  3. 如果想尽可能少的学习编程语言，可以考虑使用 js 开发后端，如 nest.js
  4. 老师授课只讲 spring，别的他不会，如果选择 2 or 3 只能靠自学

中外第六学期课程指南（大三下）

  1. UML ：开卷考试，日常实验较多，无编码 + 画图 + 撰写报告。2. 软件工程 ：开卷考试，日常有实验，较简单，编码少 + 画图 + 撰写报告。3. 微机原理与接口技术 ：开卷考试，日常有实验，汇编，编码多 + 撰写报告。4. 实训 ：共 2 个，持续 6 周，计算机软件综合实训参考数据库实训形式，微机原理实训无授课，需要到实验室进行实验（需要实验箱调试程序）。5. 自本学期起无外教课

学习指南

  1. 本学期有些同学可能要准备考研，想利用课上时间学习考研课程，自己分配好学习的时间即可。
  2. UML ：以概念、绘图为主，日常实验按照要求完成，能够知悉不同种类的图的用途，例如通信图、类图、状态机图等。建议平时不听课的同学实验时多花些时间学习 + 练习，能够极大的减轻期末压力。
  3. 软件工程 ：以概念为主，重要的是思想。软件工程的部分内容比较考验自身的实践经验，若此前已有丰富的实践经验，则在学习过程中会事半功倍，若经验较少可能会比较枯燥。内容比较宽泛，老师授课内容与课本不完全一致，但对很多案例剖析的很清晰，建议可以听一听 + 做好笔记。
  4. 微机原理 ：与计算机组成原理有相关性，主要考察汇编语言以及使用汇编编程。老师会画重点，在实验课 + 期末前多复习一下挂科很难。课程内容与期末实训强相关。
  5. 计算机软件综合实训 ：同数据库实训，建议同去年，选题可与老师商议更换，工作量大致相当即可，建议计划找工作的同学早作打算。
  6. 微机原理实训 ：主要考察使用汇编编程，选题分层次，简单的十分简单，难的也很有难度，只要课程相关的知识掌握七七八八，实训的时候花点儿时间就能搞定，但是想要拿高分还是要下一点功夫。
  7. 关于汇编 ：回旋镖来了，如果第五学期的编译原理实训中有所涉及，那么这部分会相对简单，但是还是要花时间学习，因为微机原理不仅涉及基本的指令，还涉及一些硬件接口和系统中断，课程基于 8086 汇编，实训亦使用 8086 平台进行。
  8. 关于实验 ：建议大家重视日常实验，如果上课期间想花时间学习考研知识，那就在实验课期间多补补课，对于期末复习事半功倍。
  9. UML 与软件工程 ：两者是辩证统一的，UML 可以认为是软件工程的子集，两者从内容、考核方式上均有相关性。
